## Контекст

Phase 1 — первый рабочий продукт KrinikCam. После неё пользователь должен:
1. Подключить USB камеру (Emeet Piko+ или любая UVC-совместимая)
2. Открыть приложение и увидеть превью
3. Ввести stream key YouTube
4. Нажать кнопку — и пошёл стрим

Библиотеки (по master_plan.md):
- **AndroidUSBCamera** (`jiangdongguo`) — UVC драйвер, Java/Kotlin API
- **RootEncoder** (`pedroSG94`) — RTMP клиент, аппаратный H.264 через MediaCodec

## Вопросы

### Q1: Что пользователь видит при первом запуске (до подключения USB камеры)?

Три варианта:

| Вариант | Описание |
|---------|----------|
| **A** | Onboarding-экран: "Подключи USB камеру через OTG" + картинка + кнопка "Как это сделать?" |
| **B** | Просто чёрный экран с иконкой камеры и текстом "Ожидание USB камеры…" |
| **C** | Сразу настройки: предлагает ввести stream key, пока камеры нет |

> **Ответ:** Видит фулскрин видеоискатель, который начинает выводить видео из первого доступного ему источника видео - по умолчанию это будет главная фронтальная камера устройства - если нет - селфи камера - если нет - любая подключенная камера - если нет - чёрный экран видеоискателя с сообщением о том, что нет выбранных активных источников виде. Просьба подключить устройства ввода, кнопка на соответствующую инструкцию в Settings разжеле приложения.

### Q2: Кнопка «Начать/Остановить стрим» — где и какая?

| Вариант | Описание |
|---------|----------|
| **A** | Большая розовая кнопка REC/STOP внизу экрана, всегда видна поверх превью |
| **B** | Свайп снизу → появляется панель с кнопкой и параметрами |
| **C** | Тап на превью → показывается оверлей с кнопкой, через 3 сек прячется |

> **Ответ:** нужны раскрывающиеся оверлеи поверх главного экрана, плавающие кнопки. Очень удобно это реализовано в The Sims 3 - клик по объекту, с котором хочешь взаимодействовать открыывает круговое мменю с опциями - клиен по соответствующей опции проходит дальше и открывает своё соответствующее меню с опциями или целевыми действиями.

### Q3: Как пользователь вводит ключ стрима YouTube в Phase 1?

| Вариант | Описание |
|---------|----------|
| **A** | Отдельный экран «Settings → YouTube → Stream Key» (текстовое поле) |
| **B** | Диалог при первом нажатии «Начать стрим» — приложение спрашивает ключ |
| **C** | Мини-экран настроек платформ (упрощённый StreamTargetsScreen) |

> **Ответ:** Нужен модальный диалог поверх главного экрана с настройкой стриминг платформ. Они создаются как отдельные сущности, включаются или выключаются, удаляются, редактируются. так же нужна отдельная секция в Settings все настройки из модального оверлей диалога и из секции Settings идентичны и синхронны, изменения пробрасываются из одного в другое, правят единый для них конфиг.

### Q4: Аудио в Phase 1?

Камера Emeet Piko+ имеет встроенный микрофон, но UVC-микрофон требует отдельной обработки.
Телефон (Headwolf Titan1) также имеет микрофон.

| Вариант | Описание |
|---------|----------|
| **A** | Микрофон телефона — просто и надёжно для Phase 1 |
| **B** | UVC-микрофон камеры (если есть) или телефонный (fallback) |
| **C** | Phase 1 без аудио — видео-стрим без звука, аудио в Phase 2 |

> **Ответ:** В Phase 1 можно звук с UVC камеры, но лучше уже начать реализовывать менеджер устройств. В нём UVC-микрофон - одно устройство на выбор, микрофон планшета - второе устройство на выбор. Менеджер устройств - фундаментальный модуль приложения, лучше начать его разрабатывать раньше.
Если Phase 1 тестовая, девай как проще, но модульно, с заделом на дельнейший рефактор и расширение.

### Q6: Поведение при отключении камеры во время стрима

| Вариант | Описание |
|---------|----------|
| **A** | Стрим останавливается автоматически, показывается уведомление |
| **B** | Стрим продолжается с чёрным кадром, пока камера не переподключится |
| **C** | Стрим продолжается с последним стоп-кадром (заморозка) |

> **Ответ:** Красивая заглушка с логотипом KrinikCam с подписью на основых языках мираодинаковой фразы: Please stand by, стрим скоро возобновится - чтобы подождали - эта заглушка будет стримится в поток, пока камера физически не передаёт данные в кодек.

### Q7: Scope Phase 1 — строго MVP или чуть шире?

В master_plan.md Phase 1 включает также `:data:profiles` и `:feature:profiles`
(Room DB + автодетект DeviceProfile). Это полезно, но добавляет работы.

| Вариант | Описание |
|---------|----------|
| **A** | Строго MVP: USB превью + один YouTube RTMP стрим + ввод key — **без Room DB** |
| **B** | По плану: + DataStore для хранения stream key + DeviceProfile |
| **C** | Расширенный: всё по плану + минимальный DeviceManagerScreen |

> **Ответ:** Давай по-плану, многие ФУНДАМЕНДАЛЬНЫЕ модули и блоки желательно начать реализовывать раньше. Так же ОБЯЗАТЕЛЬНО реализуй блок дебага и логирования, чтобы я мог скидывать тебе файлы логов для анализа и деббагинга.

---

### Q8: Нужен ли `:feature:codec` сканер кодеков в Phase 1?

Phase 1 можно реализовать с жёстко заданным кодеком H.264 HW.
Полный сканер MediaCodec нужен в Phase 2 (параллельные кодеки).

| Вариант | Описание |
|---------|----------|
| **A** | Phase 1: использовать H.264 HW напрямую через RootEncoder (без сканера) |
| **B** | Реализовать базовый сканер — пригодится для правильного выбора битрейта |

> **Ответ:** Важные фундаментальные модули приложения лучше начать делать раньше - фундамент, на котором будем возводить весь "небоскрёб приложения"
